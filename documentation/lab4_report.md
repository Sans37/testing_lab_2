# Лабораторная работа №4
## BDD E2E тесты аутентификации с 2FA и сменой пароля

**Дата выполнения:** 2026-03-19

## 1. Цель
Реализовать E2E‑тесты (BDD) для полной проверки аутентификации с двухфакторной проверкой и плановой сменой пароля.

## 2. Объект тестирования
Проект Nady Bakery (FastAPI + PostgreSQL). Тестируются эндпоинты `/api/v2/auth/*`.

## 3. Реализация 2FA и блокировки
- 2FA включается переменной окружения `TWO_FA_ENABLED=true`.
- При логине генерируется OTP (6‑значный код) и создается `session_id`.
- Проверка OTP: `POST /api/v2/auth/verify-2fa`.
- Лимит попыток: `AUTH_MAX_ATTEMPTS=3`, после чего аккаунт блокируется на `AUTH_LOCKOUT_SECONDS`.
- Восстановление: `POST /api/v2/auth/unlock/request` + `POST /api/v2/auth/unlock/confirm`.
- Смена пароля: `POST /api/v2/auth/request-password-change-otp` + `POST /api/v2/auth/change-password`.

## 4. BDD‑сценарии
1. **Успешный вход с 2FA**
2. **Блокировка после 3 неверных попыток + восстановление**
3. **Плановая смена пароля с OTP**

Файл feature: `src/tests/e2e/bdd/auth_2fa.feature`

## 5. Запуск тестов
Перед запуском задайте пароль технического пользователя (без хранения в коде):
```
setx E2E_TECH_USER_PASSWORD "<ваш_пароль>"
```

Запуск BDD‑тестов:
```
python -m pytest src/tests/e2e/test_auth_bdd.py -v -m e2e
```

## 6. Интеграция в CI/CD
BDD‑тесты автоматически входят в e2e‑набор и выполняются в workflow `tests.yml`.
Пароль берётся из секретов (например, `E2E_TECH_USER_PASSWORD`).

## 7. Выводы
- 2FA корректно интегрирован в процесс логина.
- Ограничение числа попыток снижает риск brute‑force.
- Восстановление учётной записи и смена пароля подтверждаются OTP.
